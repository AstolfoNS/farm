# 项目系统性重构待处理清单

## 执行原则

- 在不改变现有业务功能的基础上重构。
- 每完成一个独立步骤后运行必要检查并进行一次 Git 提交。
- 不使用全局暂存，按文件粒度暂存变更。
- 优先处理安全性、可读性、重复代码和性能风险。

## 处理顺序

### 1. 拆分 `SeedServiceImp` 职责（已完成）

目标：

- 将商店、种子类型、阶段管理等职责从 `SeedServiceImp` 中拆分出来。
- 保持 `SeedService` 对外接口不变。
- 降低单文件体量，提高后续维护和测试可读性。

验收：

- `SeedServiceImp` 不再承载所有种子业务实现细节。
- 原有接口行为保持一致。
- Maven 测试通过。
- 完成后提交 Git。

结果：

- 已抽取 `SeedViewAssembler`，承接种子列表、商店条目和选项 VO 组装职责。
- 已将成长总秒数从逐条查询优化为批量查询。
- 提交：`56e678a refactor(seed): 拆分种子视图组装职责`

### 2. 收敛前端模块重复工具函数（已完成）

目标：

- 收敛各前端模块中重复的 `escapeHtml`、`asNumber`、`toArray` 等 fallback 逻辑。
- 优先复用 `FarmUi`、`FarmAdmin`、`FarmGrid` 等已有核心工具。
- 保持页面行为和视觉风格不变。

验收：

- 重复工具函数数量减少。
- 前端模块语法检查通过。
- 涉及页面功能保持一致。
- 完成后提交 Git。

结果：

- 已在 `shop.js` 与 `store.js` 中统一使用本地 `Ui` 引用复用 `FarmUi` 工具。
- 已收敛分页渲染入口对 `FarmUi.renderButtonPager` 的直接重复访问。
- 提交：`1414d59 refactor(frontend): 收敛商店仓库工具引用`

### 3. 审查并补齐控制器 DTO 校验

目标：

- 审查 Controller 入参 DTO 的 `jakarta.validation` 覆盖情况。
- 对高风险接口补齐 `@NotNull`、`@Positive`、`@Min`、`@Size` 等约束。
- 减少 Service 层重复基础参数判空。

验收：

- 高风险写操作接口具备明确 DTO 校验。
- 错误提示清晰，不暴露内部异常细节。
- Maven 测试通过。
- 完成后提交 Git。

### 4. 审查上传文件白名单策略

目标：

- 明确是否允许任意文件上传。
- 如果业务不需要任意文件，按分类建立扩展名和内容类型白名单。
- 保持头像、种子封面、阶段资源等现有上传路径可用。

验收：

- 文件上传策略在代码中集中表达。
- 非法类型有明确错误提示。
- 路径穿越和异常扩展名风险被覆盖测试保护。
- 完成后提交 Git。
