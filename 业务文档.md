# 农场项目业务文档

## 1. 文档目标与范围

本文档用于完整说明当前农场项目的后端业务设计、接口分组、数据模型与关键规则，作为开发、联调、测试与后续迭代的统一基线。

覆盖范围：
1. 业务模块边界与职责
2. 核心业务流程（种植、收获、商店、地块经营）
3. 关键业务规则与计算口径
4. 数据表与领域对象映射
5. 接口清单与前端调用建议
6. 事务、幂等、异常处理机制
7. 配置化策略与扩展建议

不在本文档内的内容：
1. 前端页面实现细节（jQuery/EasyUI 交互代码）
2. 运维部署脚本与容器编排

---

## 2. 项目技术与实现约束

技术路线：
1. Java 21
2. Spring Boot 3 + Spring MVC
3. JPA/Hibernate（DAO 统一基于 `JpaRepository`）
4. PostgreSQL
5. 前端：jQuery + EasyUI

关键约束：
1. 不新增 `pom.xml` 依赖
2. 不使用 `javax.*` 包前缀
3. 返回体统一为 `R<T>`，分页统一 `PageResult<T>`
4. 业务完整性主要由业务层保证（当前按要求不依赖 FK/CHECK 做硬约束）
5. 全项目文本编码为 UTF-8

职责：
1. 种子类型管理（种子基础属性与经济属性）

### 3.4 作物生命周期模块（Crop Lifecycle）

职责：
1. 种植
2. 收获
3. 养护（杀虫）
4. 作物状态同步（生长中/成熟/枯萎）
5. 虫害减产与养护奖励

### 3.5 地块经营模块（Plot Management）

职责：
1. 地块解锁（顺序约束 + 金币 + 经验校验）
2. 扩地（新增地块 + 土壤经验门槛 + 金币）
3. 地块状态聚合查询
4. 地块经营流水分页（UNLOCK_PLOT/EXPAND_PLOT）
5. 地块经营业务类型字典

### 3.6 农场聚合查询模块（Farm Query）

职责：
1. 我的农场概览（地块 + 作物 + 成熟倒计时 + 可收获标记）
2. 我的种子背包 + 可种地块数

### 3.7 共性能力模块（Core）

职责：
1. 统一参数校验辅助
2. 统一错误码 + 业务异常
3. 幂等处理
4. 通用构建（资产流水/库存流水/行为日志）
5. 分页参数规范化与业务类型归一化

## 4. 核心业务流程

### 4.1 用户初始化流程

1. 创建用户基础资产（金币/经验/积分等默认值）
2. 初始化默认地块：总地块数与初始解锁数量由配置决定
3. 每块地写入 `unlock_experience_required`

### 4.2 购买种子流程（`/seed/shop/buy`）

1. 参数校验
2. `requestId` 幂等校验（用户 + 业务类型 + requestId）
3. 校验用户、种子类型
4. 计算总价并扣金币
5. 增加种子库存
6. 写资产流水（COIN/EXPENSE，biz=BUY_SEED）
7. 写库存流水（SEED/INCOME，biz=BUY_SEED）
8. 持久化幂等成功结果

### 4.3 出售果实流程（`/seed/shop/sell-fruit`）

1. 参数校验
2. `requestId` 幂等校验
3. 校验用户、果实库存
4. 扣减果实库存
5. 增加金币
6. 写库存流水（FRUIT/EXPENSE，biz=SELL_FRUIT）
7. 写资产流水（COIN/INCOME，biz=SELL_FRUIT）
8. 持久化幂等成功结果

### 4.4 种植流程（`/gameplay/plant`）

1. 校验用户、地块、种子库存
2. 校验地块已解锁且为空地
3. 校验土壤是否在种子可种范围（`enable_soil_type_bits`）
4. 扣减种子库存
5. 创建作物实例并计算时间轴（成熟时间、枯萎时间）
6. 写库存流水（SEED/EXPENSE，biz=PLANT）
7. 写作物行为日志（PLANT）

### 4.5 收获流程（`/gameplay/harvest`）

1. 校验用户、地块、作物
2. 同步作物状态，必须成熟才可收获
3. 计算本次果实收益（含虫害减产）
4. 增加果实库存
5. 增加用户经验与积分
6. 根据 `max_harvest_count` 判断是否清除作物
7. 多次收获作物按 `regrow_stage_index` 回退阶段并重置时间轴
8. 写库存流水（FRUIT/INCOME，biz=HARVEST）
9. 写作物行为日志（HARVEST）

### 4.6 养护流程（`/gameplay/care`）

1. 校验用户、地块、作物
2. 枯萎作物不可养护
3. 每次最多清除 1 只虫
4. 若实际清除了虫子，按种子配置发放金币/经验/积分奖励
5. 写资产流水（COIN/EXPERIENCE/SCORE，biz=CARE）
6. 写作物行为日志（CARE）

### 4.7 地块解锁流程（`/gameplay/plot/unlock`）

1. 校验用户、地块存在
2. 校验地块为锁定状态
3. 校验用户经验达到该地块门槛（`user_plots.unlock_experience_required`）
4. 校验顺序解锁（必须先解锁最前面的锁定地块）
5. 扣金币（按解锁策略）
6. 更新地块为解锁状态
7. 写资产流水（COIN/EXPENSE，biz=UNLOCK_PLOT）

### 4.8 扩地流程（`/gameplay/plot/expand`）

1. 校验用户
2. 选择土壤（指定 `soilTypeId` 或默认最低级土壤）
3. 校验用户经验达到该土壤门槛（`soil_types.unlock_experience_required`）
4. 计算扩地成本并扣金币
5. 创建新地块（plotIndex 递增，默认解锁）
6. 计算新地块的 `unlock_experience_required`
7. 写资产流水（COIN/EXPENSE，biz=EXPAND_PLOT）

## 5. 关键业务规则与公式

### 5.1 作物状态机

`user_crops.grow_status`：
1. `1`：生长中
2. `2`：成熟待收
3. `3`：已枯萎

状态由当前时间与时间轴字段推导：
1. `expected_ripe_at`
2. `expected_withered_at`

### 5.2 虫害减产规则

1. 基础果实数：`baseFruit = max(0, seed_types.harvest_fruit_number)`
2. 每虫减产：`lossPerBug = max(0, seed_types.fruit_loss_per_bug)`
3. 总减产：`totalLoss = bugCountBefore * lossPerBug`
4. 实际收获：`fruitGain = max(0, baseFruit - totalLoss)`

### 5.3 养护奖励规则

当且仅当本次养护实际清除虫子时发奖励：
1. 金币：`seed_types.bug_kill_coin_reward`
2. 经验：`seed_types.bug_kill_experience_reward`
3. 积分：`seed_types.bug_kill_score_reward`

### 5.4 多次收获与回生规则

1. `max_harvest_count`：最大可收获次数
2. 达到上限后作物清除
3. 未达上限则回退到 `regrow_stage_index` 并重置成熟/枯萎时间

### 5.5 地块经验门槛规则

1. 每块地独立经验门槛：`user_plots.unlock_experience_required`
2. 新增地块门槛由策略计算：
3. `baseRequiredExperience + stepTimes * requiredExperienceStep`
4. `stepTimes = max(0, plotIndex - unlockedPlotCount - 1)`

### 5.6 高级土壤经验门槛

1. 土壤经验门槛字段：`soil_types.unlock_experience_required`
2. 扩地选择高级土壤时，经验不足直接拒绝

### 5.7 成本策略规则

地块成本由配置策略计算：
1. 解锁成本：`freePlotIndexLimit/baseCostCoin/costStepCoin`
2. 扩地成本：`freePlotCountLimit/baseCostCoin/costStepCoin`

### 5.8 幂等规则

1. 关键写操作依赖 `request_idempotencies`
2. 维度：`userId + bizType + requestId`
3. 状态：`PROCESSING / SUCCESS / FAILED`
4. 重复请求优先返回缓存成功结果

## 6. 数据模型与表职责

## 6.1 基础与配置表

1. `farm.seed_qualities`：种子品质字典
2. `farm.soil_types`：土壤类型（含 bit_code、等级、经验门槛、生长倍率）
3. `farm.growth_stages`：生长阶段字典
4. `farm.seed_types`：种子类型主配置（经济、虫害、收获、回生）
5. `farm.seed_growth_stages`：种子阶段配置（阶段顺序、时长、虫害概率、展示资源）

## 6.2 用户经营数据表

1. `farm.users`：用户资产与账户信息
2. `farm.user_plots`：用户地块（土壤、锁定状态、解锁经验门槛）
3. `farm.user_crops`：地块上的作物实例（生命周期与虫害状态）
4. `farm.user_seeds`：用户种子背包
5. `farm.user_fruits`：用户果实库存

## 6.3 流水与行为记录表

1. `farm.user_asset_flows`：资产流水（COIN/SCORE/EXPERIENCE）
2. `farm.user_inventory_flows`：库存流水（SEED/FRUIT）
3. `farm.user_crop_action_logs`：作物行为日志（PLANT/CARE/HARVEST）
4. `farm.request_idempotencies`：请求幂等记录

## 9. 事务、异常与一致性策略

### 9.1 事务边界

以下写操作应在事务内执行：
1. 购买种子
2. 出售果实
3. 种植
4. 收获
5. 养护
6. 地块解锁
7. 扩地

### 9.2 异常处理

1. Service 层抛 `ServiceException` + `BizErrorCode`
2. Controller 不写业务 `try/catch`
3. 全局异常统一转 `R.failed(...)`

### 9.3 一致性原则

1. 资产变化必须有资产流水
2. 库存变化必须有库存流水
3. 作物关键行为必须有行为日志
4. 幂等写接口必须消费 `requestId`

## 11. 当前业务完成度结论

已形成完整闭环：
1. 商店闭环：买种子 -> 种植 -> 收获 -> 卖果实
2. 地块经营闭环：经验成长 -> 解锁地块 -> 扩地升级
3. 风险/收益闭环：虫害减产 -> 养护清虫 -> 奖励反馈
4. 追踪闭环：资产流水 + 库存流水 + 行为日志 + 幂等记录

现阶段可直接支撑：
1. 前端首页（总览 + 商店）
2. 种植玩法主链路
3. 经营账单与行为日志查询
